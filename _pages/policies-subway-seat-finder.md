시행일: 2026년 6월 24일

지하철 빈자리 찾기는 사용자의 개인정보를 중요하게 생각하며, 앱 서비스 제공에 필요한 최소한의 정보만 처리합니다.

## 1. 수집 및 이용하는 정보

앱은 다음 정보를 처리할 수 있습니다.

### 로그인 정보

Google 로그인을 사용할 경우 Firebase Authentication을 통해 로그인합니다.

- Firebase 사용자 식별자
- Google 계정 표시 이름
- 로그인 제공자 정보

앱 서버는 Google 계정 비밀번호를 수집하거나 저장하지 않습니다.

### 위치 정보

앱은 가까운 지하철역을 찾기 위해 기기의 위치 권한을 요청할 수 있습니다.

- 위치 정보는 가까운 역 계산에 사용됩니다.
- 정확한 위도/경도는 앱 서버에 저장하지 않습니다.
- 사용자가 위치 권한을 허용하지 않아도 앱의 일부 기능은 사용할 수 있습니다.

### 좌석 공유 정보

사용자가 좌석 정보를 공유할 때 다음 정보가 저장될 수 있습니다.

- 노선, 열차, 호차, 좌석 위치
- 내릴 역
- 공유 시각 및 만료 시각
- 도움 됐어요 수, 확인 요청 수
- 신고 사유

이 정보는 다른 이용자에게 빈자리 정보를 보여주기 위해 사용됩니다.

## 2. 정보 이용 목적

수집한 정보는 다음 목적에만 사용됩니다.

- 사용자 로그인 확인
- 가까운 지하철역 찾기
- 실시간 지하철 도착 정보 조회
- 사용자가 공유한 좌석 정보 표시
- 부정확한 정보 신고 및 품질 관리
- 서비스 안정성 유지

## 3. 제3자 제공 및 외부 서비스

앱은 서비스 제공을 위해 다음 외부 서비스를 사용합니다.

- Firebase Authentication: Google 로그인
- Cloudflare Workers 및 D1: 좌석 공유 API 및 데이터 저장
- 서울 열린데이터광장 API: 지하철 실시간 도착 및 열차 위치 정보 조회

앱은 사용자의 개인정보를 판매하지 않습니다.

## 4. 보관 및 삭제

좌석 공유 정보는 서비스 특성상 일정 시간이 지나면 만료되도록 설계되어 있습니다.

사용자는 앱의 내 정보 화면에서 로그아웃하거나 회원탈퇴를 요청할 수 있습니다.

회원탈퇴 시 Firebase Authentication 계정 삭제와 함께 서버에 저장된 사용자 식별 정보는 삭제 또는 익명화됩니다.
현재 공유 중인 좌석 정보는 삭제 상태로 변경됩니다.

로그인 관련 정보 또는 서버에 저장된 사용자 정보 삭제와 관련해 추가 문의가 필요한 경우 아래 이메일로 요청할 수 있습니다.

문의 이메일: sicorp.apps@gmail.com

## 5. 보안

앱은 개인정보와 민감한 정보를 안전하게 처리하기 위해 HTTPS 등 보안 통신을 사용합니다.

API 키와 서버 비밀키는 앱에 직접 포함하지 않고 서버 환경에서 관리합니다.

## 6. 아동 개인정보

본 앱은 아동을 대상으로 하지 않습니다. 만 14세 미만 아동의 개인정보를 고의로 수집하지 않습니다.

## 7. 개인정보처리방침 변경

개인정보처리방침이 변경될 경우 본 페이지를 통해 공지합니다.
